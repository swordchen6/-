1.默认内存管理函数的不足
利用默认的内存管理操作符 new/delete 和函数 malloc()/free() 在堆上分配和释放内存会有一些额外的开销。
首先，需要系统调用，然后根据一定的算法，到内存中寻找一块合适的内存返回。
而且内存管理函数还考虑到多线程的应用，依次，性能会进一步降低。
如果应用频繁调用分配和释放内存的函数，会产生一些外碎片，而且时间、系统开销也将会很大。

2.线程池的介绍
为了减少外碎片，减少系统开销，我们一开始就分配一片很大的内存。
线程池，从线程安全的角度来分，内存池可以分为单线程内存池和多线程内存池。
单线程内存池效率高，但安全性低。多线程内存池效率较低，但安全性高。
从内存池可分配内存单元大小来分，可以分为固定内存池和可变内存池。
固定内存池灵活性低，但效率高，可变内存池灵活性高，但效率较低。
根据具体情况的不同，内存池应有不同的变化来适应需求。

3.经典线程池的结构
（1）结构1：内存块
经典的线程池，首先选择一块固定足够大的内存块，用结构体MemBlock去管理。
如果线程池的内存块不够，可以再申请一块同样大的内存块以提供使用。
因此，为了方便内存块之间的连接，我们采用链表的结构将内存块连接。

（2）结构二：空闲的节点
将内存块分为多个节点，每个节点内保存着一个和我们想要分配的变量类型大小的内存内存小块和指向下一个节点的指针。

（3）结构三：线程池
线程池保存着内存块的起始指针，和空闲节点的起始指针。
